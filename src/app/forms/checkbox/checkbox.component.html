CHECKBOX

The materialize-checkbox component works as the native component but with Material Design styling animation and other features.

USAGE

How to use

- Import the MaterializeCheckboxModule in your module.

- Use the materialize-checkbox tag in your template.

API

Materialize Angular provides custom functions to add functionallity its components in different ways, the most common is using attributes in the component tag.

@Input Properties

Name Type Default Description
className string To specify an additional CSS class, use the "className" property. This applies to the component's first node.
disabled boolean false If true, the label and checkbox element should be displayed in a disabled state.
id string The id of the checkbox element.
indeterminate boolean false The indeterminate's attribute value of the checkbox element.
name string The name attribute of the checkbox element.
required boolean false If true, the checkbox element will be required.
value boolean false The value of the checkbox element, also this value will be set on checked attribute.

@Output Events

Name Signature Description
onChange onChangeEmitter(): EventEmitter<boolean> Event fired when the value is changed.

THEMING

Materialize Angular supports a lot of SASS variables to change all the component style so that feel to modify component look and feel by changing the following SASS variables.

The Materialize Angular SASS variable names are created base on the following pattern:

"tagClassName" + (optional "-" + "modifierTagClassName" ) + "-" + "cssProperty"
